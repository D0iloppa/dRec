# DOPL 애셋 스타일 철칙 (반드시 준수)

> 목적: 애셋을 언제 누가 추가하든 **2026-06-10 확정된 톤**이 유지되어야 한다.
> 일관성이 깨진 애셋은 검수에서 무조건 폐기한다.

## 확정 톤 (불변)
- **레트로 2000년대 초 한국 온라인게임 도트(픽셀) 포트레이트** — 큐플레이/포트리스 세대 감성
- 성숙하고 개성 있는 성인 캐릭터. **유아틱/치비/SD 금지** (NOT cute, NOT chibi, NOT childlike)
- 버스트업(머리~가슴 상단), 정면, 중앙 정렬
- 진한 외곽선 + 강한 셀 셰이딩 + 절제된 디더링, 채도 높은 색
- 캐릭터(인물) 배경은 **완전 균일한 마젠타(#FF00FF)** → 크로마키로 투명화

## 철칙
1. **이미지 생성은 반드시 `assetgen.py`를 통해서만** 한다. 임의 프롬프트로 따로 생성 금지 —
   스타일 문구의 단일 출처는 `assetgen.py`의 `STYLE` / `BASE_PROMPTS` 상수다.
2. **착용샷은 반드시 확정 베이스(`ref/base_m.png`, `ref/base_f.png`)를 레퍼런스로 첨부**해 생성한다.
   (얼굴·비율·헤어가 베이스와 달라진 결과물은 폐기 후 재생성)
3. **`ref/`의 베이스는 사실상 불변 자산**이다. 교체하면 기존 전체 착용샷과 톤이 어긋나므로,
   교체 시엔 모든 아이템 착용샷을 전부 재생성해야 한다 (`all --force`).
4. 신규 아이템 프롬프트는 `items.json`에 추가하고, 본문은 **"Change ONLY this: ..." 패턴**을 유지한다.
   아이템 묘사에 스타일 형용사(cute, chibi 등)를 섞지 않는다 — 스타일은 STYLE 상수가 책임진다.
5. 배경은 항상 "perfectly flat uniform magenta (#FF00FF), no dithering/noise" 문구를 유지하고,
   투명화는 `chroma_key`(테두리 flood-fill)로 한다. 수동 포토샵 보정 금지(재현 불가).
6. 산출물은 `assets_gen/`(검수 대기) → 통과분만 `apps/client/public/avatar/<m|f>/`로 승격한다.
   검수 기준: ① 베이스와 같은 인물인가 ② 지시한 아이템만 바뀌었는가 ③ 배경이 깨끗이 투명한가.
7. 모델을 바꾸면(예: nano banana pro) 반드시 베이스 1쌍을 먼저 시험 생성해 기존 ref와 비교 검수한 뒤 진행한다.

## v2 레이어 구조 철칙 (2026-06-10 추가 — 현행 표준)
- 톤 레퍼런스는 `ref/tone.png`(큐플레이 아바타 스프라이트). **base는 "아무것도 안 입은 상태"**
  (남: 맨몸, 여: 회색 반두) — 아이템은 전부 overlay로 위에 합성된다.
- base는 성별당 3종(`bases` 명령, 시트 1요청→3분할). **`sheet_base.png`는 불변** —
  overlay 정렬의 기준 캔버스이므로 교체 시 모든 overlay 재생성 필요.
- **아이템 overlay는 base마다 1장** (`overlay <code>`): "입힌 모습으로 편집" 생성 후
  base와의 **픽셀 diff로 아이템 레이어만 추출**한다. "아이템만 그려라" 방식은 정렬·품질이
  나빠 금지. 시트에서 누락된 패널은 단일 base 폴백이 자동 처리.
- 검수는 `compose <baseN> <code...>`로 **여러 아이템을 겹쳐서** 확인한다 (조합이 핵심).
- 신규 base 추가 = 기존 모든 아이템의 overlay를 그 base에 대해 재생성해야 함을 의미한다.

## 캐릭터 페르소나 (2026-06-10 확정 — personas.json이 원본)
base는 단순 외형 변형이 아니라 **고정 캐릭터**다. 향후 모든 파생 애셋(표정, 포즈, 홍보 일러스트)은
해당 캐릭터의 look + persona 문구를 프롬프트에 함께 넣어 일관성을 유지한다.

| base | 이름 | 한 줄 페르소나 |
|------|------|----------------|
| m/b1 | 지호 (Jiho) | 도시의 승부사. 차갑게 계산하고, 웃으면서 이긴다. |
| m/b2 | 태오 (Taeo) | 타고난 분위기 메이커. 지는 게임도 즐겁게 만든다. |
| m/b3 | 강우 (Kangwoo) | 말수 적은 승부 본능. 표정은 없지만 손은 빠르다. |
| f/b1 | 세라 (Sera) | 시크한 전략가. 조용히 보다가 정확히 찌르고, 마지막에 웃는다. |
| f/b2 | 유나 (Yuna) | 햇살 같은 인싸. 리액션 부자, 꾸미기에 진심. |
| f/b3 | 민지 (Minji) | 쿨한 관찰자. 조용히 정답만 말한다. |

- 캐릭터별 파생 생성 시: `personas.json`의 `look` + `persona`를 프롬프트에 포함 + 해당 base 이미지를 레퍼런스 첨부.
- 이름/페르소나 변경은 personas.json에서만 하고, 클라이언트 표기(avatarRender.ts BASE_NAMES)와 동기화한다.
